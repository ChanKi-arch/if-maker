# IF Maker — Alchemy Lab

> **Pick materials. Set ratios. Pick a form. Explore 5 ways they could combine.**
> Then decompose the result back into its parts. Open-source, deterministic, LLM-optional.

```
Pick → Weigh → Shape → Explore → Commit → Decompose
```

IF Maker is a virtual lab that lets you **combine materials, objects, and elements into new speculative concepts** — and, crucially, **decompose any result back into its parts**. Every synthesis is a typed object with a structural diagram, weighted properties, and an Unity-ready export.

It's a prototype built in a few days with [Claude Code](https://claude.ai/code), FastAPI, Next.js, and Three.js.

---

## What's different

Most AI tools treat mixing as **text-in / text-out**. IF Maker treats it as a **structural operation on parts**:

- **Structural merge** — `Chair (seat/backrest/legs/frame) + Carbon Fiber (woven/resin)` doesn't become a blob of text. It becomes a 3D concept whose parts you can still see, label, and pick apart.
- **Parts balancing** — If A has 4 parts and B has 1, B is cycled to 4 so both sides contribute equally to the merge.
- **Reversible decompose** — Every synthesis remembers its source items. `⟠ DECOMPOSE` is a real inverse, not a regeneration.
- **Percentage weights** — Move sliders to pick a 70/30 mix. Each part's contribution and every numeric property scales with the ratio.
- **Form morphology** — Re-shape the whole concept as `Solid / Diffuse / Condensed / Fibrous / Plate / Porous`. Different mode = different properties, different feel.
- **5-strategy Explore** — Same inputs, same ratio, **five fundamentally different ways to combine**: Shell Coat, Core Infuse, Layer Stack, Selective Swap, Homogeneous Blend. Each interprets the ratio differently.
- **Element transforms** — `🔥 Fire + 🪑 Chair = Burnt Chair` with actual property changes and a red-glowing 3D model.
- **Chain synthesis** — A synthesized concept is a first-class library item. Mix it back in with anything else.

No ML training. No vector search. No database. No auth. Just a pure functional engine you can reason about.

## Quick start

### Docker Compose (recommended)

```bash
git clone https://github.com/your-name/if-maker.git
cd if-maker
docker compose up -d
```

Open <http://localhost:3000>. Backend on `8000`.

### Local dev

**Backend** (FastAPI):

```bash
cd backend
python -m venv .venv && source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```

**Frontend** (Next.js 14):

```bash
cd frontend
npm install
npm run dev
```

Open <http://localhost:3000>.

## Optional LLM setup (BYOK)

IF Maker works fully without any LLM — every result is generated by a deterministic rule engine.

To enable optional LLM enhancement (creative naming, richer summaries, smart material creation), copy `.env.example` to `backend/.env` and fill in **your** API key:

```env
LLM_API_KEY=sk-...
LLM_BASE_URL=https://api.openai.com/v1
LLM_MODEL_NAME=gpt-4o-mini

# Optional tuning
LLM_TIMEOUT=15
LLM_MAX_RETRIES=1
```

Any OpenAI SDK-compatible endpoint works:

| Provider | Base URL | Sample model |
|---|---|---|
| **LM Studio** (local, recommended) | `http://localhost:1234/v1` | `qwen2.5-7b-instruct` |
| Ollama | `http://localhost:11434/v1` | `llama3.1` |
| OpenAI | `https://api.openai.com/v1` | `gpt-4o-mini` |
| DeepSeek | `https://api.deepseek.com/v1` | `deepseek-chat` |
| Groq | `https://api.groq.com/openai/v1` | `llama-3.1-70b-versatile` |
| Alibaba Qwen | `https://dashscope.aliyuncs.com/compatible-mode/v1` | `qwen-plus` |

The header shows `◉ sLLM` when the LLM is online. All calls are wrapped with a 15s timeout and logged.

## Example workflows

### Design a new material for an indie game

1. Search `chair`, click `+ add` → slot 1
2. Search `carbon fiber`, click `+ add` → slot 2
3. Chair slider → 60%, Carbon Fiber slider → 40%
4. Form mode → `🧶 Fibrous`
5. Click `🎲 EXPLORE` — pick `💧 Core Infuse` (2 of 4 parts infused)
6. `SELECT →` commits the variant
7. `📦 EXPORT → Unity ScriptableObject` downloads the recipe JSON

### Reverse-engineer a concept

1. Synthesize something: `Drone + Titanium + Fire`
2. `⟠ DECOMPOSE` — slots repopulate with the exact source items
3. Tweak ratios, try a different form mode, synthesize again

### Create a material that doesn't exist yet

1. Type `aerogel foam` in search
2. If no match, click `+ CREATE NEW: aerogel foam` (auto-appears when query is unique)
3. The heuristic (or LLM if enabled) generates group, properties, structure
4. It's now a first-class library item — mix it with anything

## API

Backend lives at `http://localhost:8000`.

```http
GET  /                       status
GET  /items/search?q=        search library (materials + objects + elements + user-created)
GET  /items/groups           list of categories
GET  /items/{id}             full item detail
POST /items/create           { name, hint? } → dynamically generated item
POST /mix/generate           { item_ids, goal, ratios?, form_mode?, enhance? }
POST /mix/explore            returns 5 strategy variants for the same inputs
POST /mix/decompose          { concept_name } → suggested source components
GET  /mix/llm-status         { enabled, model }
POST /export/unity           Unity ScriptableObject JSON download
POST /export/json            generic JSON
POST /export/csv             flat rows
GET  /export/formats         list of supported export formats
```

## Intentionally out of scope

❌ User accounts / auth
❌ Database (filesystem JSON only)
❌ Real CAD or 3D modeling
❌ Scientific simulation (no FEM, DFT, chemistry solver)
❌ Multi-user collaboration
❌ Vector search

These are **chosen constraints**, not missing features. v0 proves the **Pick → Weigh → Shape → Explore → Commit → Decompose** loop with the smallest possible surface area.
